# Что можно сделать со сайтом перед деплоем

Краткий чеклист, чтобы архив выглядел нормально при выкладке на свой хост.

---

## Уже сделано

- **Подмена домена** — в HTML все ссылки `androidchity.ru` подменяются на текущий хост (при запуске через `node server.js`). Локально всё открывается по `http://127.0.0.1:8080`.
- **Картинки** — недостающие изображения закрыты заглушками (серые плитки с текстом). Часть страниц уже с картинками из Hugging Face.
- **Пуш-уведомления** — запрос разрешения отключён скриптом в HTML.
- **Категории** — на страницах категорий карточки постов отображаются (подключены `archive-fix.css` и скрипт).
- **Прокси картинок** — запросы к `/wp-content/uploads/...` при отсутствии файла проксируются с оригинального сайта (если он ещё отдаёт).

---

## Что сделать перед/при деплое

### 1. Домен и протокол

- При деплое **с Node-сервером** достаточно указать свой домен; подмена по `Host` даст нужные ссылки. Для HTTPS поднять сервер за reverse proxy (nginx/Caddy) с SSL.
- При **статическом деплое** (Netlify, GitHub Pages, nginx только статика) — один раз заменить в файлах `androidchity.ru` на свой домен (или оставить скрипт преддеплоя, который подставляет домен из переменной окружения).

### 2. Единая страница 404

- Сейчас 404 отдаёт простой HTML из `server.js`. При статике нужен свой `404.html` в корне (и поддержка его хостингом). Можно скопировать разметку главной и заменить контент на «Страница не найдена» и ссылку на главную.

### 3. Favicon и meta

- Проверить, что в корне есть `favicon.ico` (или ссылка в HTML ведёт на существующий файл).
- При желании добавить в `<head>` главной и шаблонов мета-теги: `description`, `og:image` для соцсетей.

### 4. Sitemap для поисковиков

- Сгенерировать `sitemap.xml` по списку восстановленных URL (например, из `URL_REPORT.json` или по обходу папок). Указать в нём уже **итоговый** домен деплоя.

### 5. Редиректы

- Решить: с `www` на без `www` или наоборот. Настроить редирект в nginx/Cloudflare/панели хостинга.

### 6. Кэширование статики

- Для CSS/JS/картинок выдать заголовки `Cache-Control` (например, `max-age=86400`). В nginx — по расширению или по префиксу путей (`/wp-content/`, `/.s/`).

### 7. Отключение лишнего

- В HTML могут остаться скрипты аналитики/рекламы с битых доменов. Поиск по `script src=` и отключение или замена на пустые заглушки уменьшит ошибки в консоли и ускорит загрузку.

### 8. Подпись «архив»

- На главной или в футере можно добавить текст вроде: «Архив сайта androidchity.ru по состоянию на [дата]. Только чтение.» — чтобы было понятно, что это копия.

### 9. Проверка битых ссылок

- После подмены домена прогнать сайт линк-чекером (например, по списку из `URL_REPORT.json` или по sitemap). Исправить или удалить ссылки на несуществующие страницы (например, на авторов).

### 10. Роботы и индексация

- Если архив не нужно индексировать — положить в корень `robots.txt` с `Disallow: /`.
- Если нужно — в `robots.txt` указать путь к `sitemap.xml` с итоговым доменом.

## Проверка страниц локально

Список из твоего сообщения проверен скриптом `scripts/check-pages.js`: **все 142 уникальных URL** по адресу **http://127.0.0.1:8080** открываются (файлы или папки с `index.html` есть). Полный список локальных ссылок скрипт выводит в консоль (раздел «Локальные ссылки (работают)»).

Перед деплоем: запусти `node server.js`, открой несколько ссылок из вывода — убедись, что стили и картинки подгружаются и домен в ссылках локальный.
