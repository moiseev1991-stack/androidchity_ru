# Восстановленный сайт androidchity.ru (локальный архив)

Локальная копия сайта из веб-архива. Запуск с вашей машины — через локальный HTTP-сервер.

## Как запустить

### Вариант 1: Двойной щелчок (проще всего)

1. Установите [Node.js](https://nodejs.org/), если ещё не установлен.
2. Запустите **RUN.bat** (двойной щелчок по файлу в папке проекта).
3. Через пару секунд откроется браузер. Окно с сервером не закрывайте.

### Вариант 2: Из терминала

1. Установите [Node.js](https://nodejs.org/).
2. Закройте все старые окна, где уже запускали `node server.js` (или выполните: `taskkill /F /IM node.exe`).
3. В папке проекта выполните:
   ```bash
   node server.js
   ```
4. Откройте в браузере: **http://127.0.0.1:8080**
5. Проверка: **http://127.0.0.1:8080/ok** должна вернуть «OK».

Сервер подменяет в HTML ссылки `androidchity.ru` на текущий хост, поэтому стили и переходы по сайту работают локально.

### Вариант 2: Другой порт

```bash
set PORT=3000
node server.js
```
Сайт будет доступен по адресу http://localhost:3000

### Вариант 3: Python (если нет Node.js)

```bash
# Python 3
python -m http.server 8080
```
Откройте http://localhost:8080  
Примечание: при использовании Python подмена хоста в HTML не выполняется — ссылки и стили могут вести на оригинальный домен.

## Структура

- **/** — главная (редирект на ленту записей).
- **/page/2/**, **/page/3/**, … — страницы ленты.
- **/2024/12/21/slug/** — отдельные записи (статьи).
- **/category/…** — категории.
- **/download/** — страница загрузки.
- **/search/…** — страницы поиска (по сохранённым запросам).

Дизайн и вёрстка взяты из сохранённых HTML; стили и скрипты подгружаются из папок `wp-content`, `.s` и т.д. При запуске через `node server.js` ссылки на домен заменяются на текущий хост.
